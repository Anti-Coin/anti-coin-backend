## 2. 문서 역할
1. `PROJECT_IDENTITY`: 비전/목적/철학/비목표
2. `ENGINEERING_CONSTITUTION`: 구현/검증/변경에 대한 헌장
3. `OPERATING_CONSTRAINTS`: 운영 현실과 제약
4. `DECISIONS`: 의사결정 이력(ADR-lite)
5. `PLAN_LIVING_HYBRID`: 변경 가능한 로드맵
6. `TASKS_MINIMUM_UNITS`: 최소 단위 실행 보드
7. `TECH_DEBT_REGISTER`: 기술 부채 레지스터
8. `TEST_STRATEGY`: 테스트 철학과 매트릭스
9. `SESSION_HANDOFF`: 다음 세션 인수인계 요약
10. `GLOSSARY`: 용어 정의
11. `SESSION_BOOTSTRAP_PROMPT`: 새 세션 시작용 프롬프트 템플릿

## 3. 갱신 규칙
1. 정책/우선순위가 바뀌면 `DECISIONS` 먼저 갱신
2. 계획이 바뀌면 `PLAN`과 `TASKS` 동시 갱신
3. 구현 후 새 부채가 생기면 `TECH_DEBT_REGISTER` 기록
4. 세션 종료 시 `SESSION_HANDOFF`를 최소 3줄 이상 업데이트
